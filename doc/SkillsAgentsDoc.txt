Skills, Agents & wann was entsteht
Zusatzmaterial zu Woche 3 — AI-Augmented PM


1. DIE DREI EBENEN VON CLAUDE CODE

Claude Code kennt drei Ebenen, die aufeinander aufbauen:

Prinzipien  |  CLAUDE.md  |  Immer aktiv — wird bei jedem Start geladen
Skills      |  .claude/skills/name/SKILL.md  |  Auf Abruf — du rufst /name auf
Agents      |  .claude/agents/name.md  |  Autonom — läuft selbstständig, orchestriert andere

Die Faustregel: Je weiter unten in der Tabelle, desto mehr Eigenständigkeit — und desto mehr Struktur brauchst du vorher.


2. DER ENTSTEHUNGSWEG: VON EXPERIMENT BIS AGENT

Der praktische Weg in der Arbeit sieht fast immer so aus:


Stufe 1: Experiment im Chat

Du probierst etwas aus. Du gibst Claude einen Prompt im Chat, schaust was rauskommt, verfeinerst ihn. Kein Dokument, kein Setup.

Wann bleibst du hier?
- Du weißt noch nicht ob die Aufgabe wiederkehrt.
- Du willst das Ergebnis sehen bevor du investierst.
- Es ist ein Einmalprozess.


Stufe 2: Prompt in einer einfachen MD-Datei

Du merkst: der Prompt ist gut, du willst ihn nicht verlieren. Du speicherst ihn in einer Markdown-Datei — als Notiz, als Vorlage, als Gedächtnis.

Wann macht das Sinn?
- Du oder dein Team nutzt den Prompt regelmäßig, aber manuell.
- Du willst ihn versionieren oder teilen.
- Er braucht noch kein eigenes Verhalten, nur Wiederverwendbarkeit.

Eine einfache .md-Datei ist kein Skill. Claude liest sie passiv wenn du sie ihm gibst — sie wird nicht automatisch ausgeführt.


Stufe 3: Skill

Du machst aus dem Prompt einen Skill — eine strukturierte SKILL.md unter .claude/skills/name/. Ab jetzt ist der Prompt per /name direkt aufrufbar. Claude weiß automatisch dass er existiert.

Wann wird aus einer MD-Datei ein Skill?
- Die Aufgabe ist wiederkehrend und immer gleich strukturiert.
- Du willst einen festen Output: immer dasselbe Format, dieselbe Logik.
- Du willst den Prozess einschränken: welche Tools darf Claude nutzen? (allowed-tools)
- Andere sollen ihn nutzen können ohne den Prompt zu kennen.

Ein Skill ist im Wesentlichen: ein Prompt mit Frontmatter und einem festen Aufrufpfad.


Stufe 4: Agent

Ein Agent ist ein Skill, der nicht wartet bis du ihn rufst — er handelt. Er orchestriert andere Skills, entscheidet selbst welche Tools er nutzt, und läuft einen Prozess komplett durch.

Wann wird aus einem Skill ein Agent?
- Es gibt mehrere Schritte, die immer in derselben Reihenfolge ablaufen.
- Schritt 2 hängt vom Ergebnis von Schritt 1 ab.
- Du willst das ganze ohne manuelle Eingriffe durchlaufen lassen.
- Die Aufgabe ist komplex genug dass du einen "Verantwortlichen" brauchst, nicht nur einen "Helfer".

Ein Agent in Claude Code ist eine .md-Datei unter .claude/agents/ mit tools-Frontmatter. Claude startet ihn als eigenständigen Subprozess.


3. ENTSCHEIDUNGSBAUM

Ist die Aufgabe einmalig?
  → Ja  → Im Chat erledigen. Kein Dokument nötig.
  → Nein ↓

Wird der Prompt öfter gebraucht, aber ohne festes Verhalten?
  → Ja  → Als einfache .md-Datei speichern (Vorlage/Referenz).
  → Nein ↓

Hat die Aufgabe immer dieselbe Struktur und denselben Output?
  → Ja  → Skill bauen (.claude/skills/name/SKILL.md)
  → Nein → Weiter verfeinern, dann nochmal fragen.

Besteht die Aufgabe aus mehreren Schritten, die automatisch ablaufen sollen?
  → Ja  → Agent bauen (.claude/agents/name.md)
  → Nein → Ein Skill reicht.


4. WAS IST EIN SKILL — GENAUER

Ein Skill ist eine Markdown-Datei mit zwei Teilen:

Frontmatter (zwischen ---):
- allowed-tools — welche Tools der Skill nutzen darf (z.B. Read, Write)
- disable-model-invocation — ob Claude ihn direkt ausführt oder nur als Vorlage liest

Prompt-Inhalt:
- Anweisungen, Vorgehen, Output-Format
- Alles was Claude beim Ausführen des Skills lesen soll

Der Ordnername wird automatisch zum Befehl: feedback-synthesizer/SKILL.md → /feedback-synthesizer. Skills sind Teil des Projekts — versionierbar, teilbar, reproduzierbar.
