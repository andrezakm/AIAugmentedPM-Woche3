Remote Interfaces & Computer Use — der aktuelle Stand
Zusatzmaterial zu Woche 3 — AI-Augmented PM

Stand: März 2026


ÜBERBLICK: DREI NEUE WEGE CLAUDE ZU STEUERN

In den letzten Monaten sind drei Arten von Remote-Interfaces hinzugekommen — auch darin unterschiedlich, wo sie laufen:

Channels              |  Nachrichten-Apps als Steuerkanal        |  Claude Code CLI     |  Pro, Max, Teams (mit Admin-Freigabe)
Remote Control        |  Laufende Session überwachen/eingreifen  |  Claude Code CLI     |  Pro, Max, Teams
Dispatch + Comp. Use  |  Aufgaben von unterwegs, Apps bedienen   |  Claude Desktop App  |  Pro, Max — noch nicht Teams/Enterprise


1. CHANNELS — TELEGRAM, DISCORD, IMESSAGE

Channels sind Nachrichten-Apps als Eingangskanal für Claude Code. Du schreibst eine Nachricht in Telegram oder Discord — Claude Code in deiner laufenden Session empfängt sie, führt die Aufgabe aus und antwortet zurück.

Unterstützte Kanäle (Stand März 2026):
- Telegram — vollständig integriert
- Discord — vollständig integriert
- iMessage — vollständig integriert (nur macOS, liest direkt aus der Messages-Datenbank)
- Fakechat — lokaler Testkanal für Entwicklung

Kein Slack, kein WhatsApp — Community hat beides angefragt, aber noch nicht verfügbar.

Wie es funktioniert:
Ein Channel ist technisch ein MCP-Server, der Events in eine laufende lokale Session pusht. Das heißt: Claude Code muss offen sein, damit Nachrichten ankommen. Für "immer erreichbar" läuft Claude Code als Hintergrundprozess.

Einrichtung:
- Telegram/Discord: Pairing-Code beim ersten Start
- iMessage: automatisch über Self-Chat, dann Kontakte per Handle hinzufügen
- Teams/Enterprise: Admin muss channelsEnabled in den Managed Settings aktivieren

Sicherheit:
- Sender-Allowlist: nur freigegebene IDs können Nachrichten senden, alle anderen werden still ignoriert
- Permission Relay: Claude kann Freigabe-Anfragen zurück in den Channel schicken — du genehmigst vom Handy

Quelle: https://code.claude.com/docs/en/channels


2. REMOTE CONTROL (Claude Code CLI)

Remote Control läuft in Claude Code CLI: du öffnest claude.ai im Browser oder in der mobilen App und greifst auf eine laufende Claude Code Session zu — ohne Messenger-App dazwischen.

Wann sinnvoll:
- Du bist nicht am Terminal, willst aber eine laufende Aufgabe überwachen oder eingreifen.
- Einfacher als einen Channel einzurichten.

Quelle: https://code.claude.com/docs/en/remote-control


3. DISPATCH + COMPUTER USE (nur Claude Desktop App)

Dispatch und Computer Use sind Features der Claude Desktop App — sie laufen nicht in Claude Code CLI.

Dispatch ist das Grundprinzip: du schickst von der Claude-App auf dem Handy oder von claude.ai eine neue Aufgabe an Claude Desktop. Claude Desktop startet eine Session und führt die Aufgabe aus. Du musst nicht am Rechner sitzen.

Computer Use erweitert Dispatch: Claude kann dabei nicht nur Text verarbeiten, sondern native Desktop-Apps direkt bedienen — klicken, tippen, scrollen, Tastenkürzel verwenden. In jeder App die keine API oder CLI hat.

Verfügbarkeit:
- Nur macOS
- Nur Pro und Max — Teams/Enterprise noch nicht verfügbar (Stand März 2026, "rolling out shortly")
- Benötigt Claude Desktop App lokal

Benötigte macOS-Berechtigungen:
- Bedienungshilfen (Accessibility) — für Klicken, Tippen, Scrollen
- Bildschirmaufnahme (Screen Recording) — für Screenshots

Wie Claude vorgeht (in dieser Reihenfolge):
1. Connector / MCP    — wenn verfügbar, schnellster Weg
2. Bash               — für alles was per Shell geht
3. Chrome Extension   — für Browseraufgaben
4. Computer Use       — Fallback, langsamer aber für alles

Claude greift auf Computer Use erst zurück wenn alle anderen Wege nicht funktionieren.

Zugriffsstufen je App:
Nur anschauen    →  Screenshots               →  Browser, Trading-Plattformen
Nur klicken      →  Klicken, Scrollen         →  Terminal, IDEs
Volle Kontrolle  →  Klicken, Tippen, Drag     →  Alles andere

Kombination mit Channels:
Channels und Dispatch können kombiniert werden: Nachricht an Claude über Telegram → Claude Desktop führt Aufgaben aus, auch mit App-Steuerung → Ergebnis kommt zurück in den Chat.

Ehrlicher Hinweis von Anthropic: Computer Use ist noch früh. Claude kann Fehler machen. Nicht für kritische oder irreversible Aktionen ohne menschliche Überprüfung verwenden.

Quellen:
https://code.claude.com/docs/en/desktop
https://claude.com/blog/dispatch-and-computer-use
https://support.claude.com/en/articles/13947068-assign-tasks-to-claude-from-anywhere-in-cowork


DIE WICHTIGSTE UNTERSCHEIDUNG

Channels       =  Nachrichten-App als Eingangskanal → läuft in Claude Code CLI
Remote Control =  Browser/Handy als Fenster in eine laufende Session → läuft in Claude Code CLI
Dispatch       =  neue Aufgabe von unterwegs an Claude Desktop schicken → nur Desktop App
Computer Use   =  Erweiterung von Dispatch, Claude bedient dabei auch native Apps → nur Desktop App

Channels und Dispatch können kombiniert werden. Remote Control und Dispatch lösen ähnliche Probleme auf verschiedenen Wegen.


DER MERKSATZ

Channels bringen Nachrichten rein (CLI). Remote Control gibt dir Sicht (CLI). Dispatch schickt Aufgaben an den Desktop. Computer Use gibt Claude dabei Hände.


WANN HILFT WAS — KONKRETE SZENARIEN

Channels (Telegram / Discord / iMessage)
- Du bist im Meeting und willst Claude eine Recherche starten lassen — Nachricht ins Telegram, Claude arbeitet auf deinem Rechner weiter.
- Du willst ein tägliches Briefing auslösen ohne den Laptop aufzumachen — einfach morgens eine Nachricht schicken.
- Dein Team nutzt Discord — jedes Mitglied kann Aufgaben an Claude schicken, ohne Claude Code selbst zu kennen.

Remote Control
- Du hast eine lange Analyse gestartet und willst vom Handy aus prüfen ob Claude noch arbeitet oder hängt.
- Du willst eine laufende Session nicht abbrechen, aber kurz eingreifen oder eine Folgefrage stellen — ohne neue Session zu öffnen.

Dispatch (Desktop App)
- Du bist unterwegs, fällt dir eine Aufgabe ein — Dispatch schickt sie an deinen Desktop, Claude erledigt sie bis du zurück bist.
- Du willst Claude eine mehrstündige Aufgabe starten lassen ohne selbst am Rechner zu sitzen.

Computer Use (Erweiterung von Dispatch)
- Ein Tool hat keine API — z.B. ein internes Web-Interface oder eine Desktop-App. Claude navigiert selbst darin.
- Du willst Screenshots aus einem Tool ziehen, das keinen Export hat — Claude macht das visuell.
- Repetitive GUI-Aufgaben die sich nicht automatisieren lassen: Formulare ausfüllen, Daten aus Masken kopieren, Statusprüfungen in Legacy-Software.
